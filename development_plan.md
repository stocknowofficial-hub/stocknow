# 📈 Stock Now 리뉴얼 개발 계획서 (Membership & Payment Automation)

## 1. 🎯 개요 (Overview)
기존의 강력하고 신뢰성 높은 **텔레그램(Telegram) 알림 시스템은 그대로 유지**하되, 수동으로 처리되던 **회원 관리 및 결제 시스템을 웹 기반으로 100% 자동화**하는 것이 핵심 목표입니다. 
구글, 네이버, 카카오 등 소셜 로그인을 통해 회원을 식별하고, Payapp 결제 연동을 통해 한국장(KR) / 미국장(US) 구독 권한을 자동으로 제어합니다.

---

## 2. 🚀 주요 변경 사항 (Key Changes)

### A. 소셜 로그인 기반 웹 멤버십 구축 (Google, Naver, Kakao)
- 텔레그램 계정과 연동할 수 있는 웹 기반의 회원가입/로그인 페이지 구축.
- 사용자는 소셜 로그인(OAuth)을 통해 간편하게 접속하여 자신의 구독 상태를 확인 및 관리.

### B. 결제 시스템 자동화 (Postype -> Payapp)
- 수동 확인이 필요했던 포스타입에서 **Payapp API 기반의 자동 결제 시스템**으로 전환.
- 웹페이지 내에서 상품(한국장 전용, 미국장 전용, 통합권 등)을 선택하고 결제 진행.
- 결제 완료 시 서버 훅(Webhook)을 통해 DB에 구독 기간 및 권한 자동 반영.

### C. 텔레그램 봇(API)을 통한 자동 초대 및 강퇴 (KR/US 분리)
- 텔레그램 채널/봇의 알림 기능은 기존의 장점을 살려 그대로 활용.
- **권한 연동**: 웹에서 결제가 완료되면, 백엔드가 텔레그램 봇 API를 호출하여 구독한 마켓(KR/US)의 전용 채널 또는 VIP 그룹에 **자동 초대(초대 링크 발송 또는 직접 추가)**.
- **구독 만료 관리**: 구독 기간이 종료되면 봇이 자동으로 채널에서 **강퇴(Kick)** 처리하여 권한 회수.

### D. 완전 이원화된 하이브리드 아키텍처 (Cloudflare + Local)
- **Local (기존 유지)**: `watcher`(한투 API 시장 감시 & Redis 송신), `worker`(Gemini AI 분석 & 텔레그램 메시지 발송)는 기존 로직을 100% 유지하여 강력한 알림 기능 제공.
- **Cloudflare (신규 도입)**: 사용자 정보 관리, 웹페이지 표시, 소셜 로그인 연동, 결제 확장 (Payapp) 등 **모든 유저 관리 로직**을 Cloudflare Pages와 Workers(D1 DB 포함)로 이관하여 서버리스(Serverless)로 무한 확장성 확보.

## 3. 🏗️ 아키텍처 변경 대조 (As-Is vs To-Be)

### 🔹 As-Is (현재)
- **정보 전달**: Watcher -> 텔레그램 봇 (종목 정보 푸시)
- **결제 및 회원 관리**: 
  1. 관리자가 포스타입에서 결제 수동 확인.
  2. 텔레그램 시크릿 채널 링크 수동 생성 및 전달.
  3. 만료 시 수동 강퇴 처리.

### 🔹 To-Be (변경 후: 완벽한 자동화 파이프라인)
- **Core 엔진 (Local)**:
  - `watcher`: 한투 API 주식 현황 감지 -> Redis 송신.
  - `worker`: 감지된 정보 기반 Gemini AI 분석 -> Telegram 채널/그룹 발송.
- **가입 & 결제 엔진 (Cloudflare Pages/Workers 기반)**:
  1. 유저가 **Cloudflare 웹페이지** 접속 후 **소셜 로그인** (플랫폼 신뢰도 확보).
  -> 꼭 소셜 로그인안하고 결제연동해도되면 그대로 연동해도됨! 이거는 같이 생각해보자.
  2. 내 정보 탭에서 **텔레그램 계정 연동(app_id <-> chat_id 매핑)**.
  3. **Payapp 결제** 완료 시 Cloudflare Worker가 Webhook 수신, D1 DB 반영.
  4. 웹 백엔드(Worker)가 텔레그램 Bot API를 호출해 유저를 KR/US 전용 채널에 **자동 초대** 발송.
  5. 1달 만료 시, Worker Cron 트리거가 유저를 채널에서 **자동 강퇴** 후, 결제 독려 URL 전송.

## 4. 📅 단계별 개발 마일스톤 (Milestones)

### Phase 1: 웹 프론트엔드 및 소셜 로그인 연동 (Next.js)
- [ ] 신규 Cloudflare 계정(`stocknow.official`) 설정 및 D1 (사용자 DB) 구축.
- [ ] Next.js 기반 반응형 웹 프론트엔드(`frontend_web`) 세팅 및 로고 적용.
- [ ] OAuth (Google, Kakao, Naver) 소셜 로그인 기능 연동.
- [ ] 유저 마이페이지 구현 (내 구독 상태 확인, 텔레그램 연동 딥링크 발급).

### Phase 2: Payapp 결제 연동 및 권한 분기
- [ ] Cloudflare Workers API에 Payapp 결제 성공 Webhook 엔드포인트 구축.
- [ ] 결제 시 한국장(KR) / 미국장(US) 권한 D1 DB 자동 반영 로직 개발.
- [ ] 웹 프론트엔드 마이페이지: 내 구독 만료일 확인 및 결제 UI (확장성을 고려한 컴포넌트 구조).

### Phase 3: 텔레그램 Bot 연동 (초대 & 강퇴 자동화)
- [ ] [Cloudflare Worker] 결제 완료 시 텔레그램 Bot API를 통해 유저에게 채널 1회용 자동 초대 링크 발송.
- [ ] [Cloudflare Worker Cron] 매일 자정 D1 DB 검사 후, 만료된 유저를 텔레그램 채널에서 자동 Kick(강퇴).
- [ ] 강퇴 시 텔레그램 봇으로 "구독이 만료되었습니다. 결제 후 다시 이용해주세요 [웹 URL]" 안내 메시지 자동 발송 로직 추가.

### Phase 4: 기존 Core(Watcher/Worker)와 시너지 테스트
- [ ] Local의 Watcher/Worker가 정상적으로 종목을 쏴주는지 검증 (기존 로직 침범 없음).
- [ ] 가입 -> 로그인 -> 결제 -> 텔레그램 초대 -> 1달 만료 -> 채널 강퇴 -> 재유도 메시지 전체 통합 시나리오 테스트.
- [ ] 정식 서비스 오픈 및 기존 회원 마이그레이션.
